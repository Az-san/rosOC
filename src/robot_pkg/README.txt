＜フォルダ＞
・data-----------------------ロボット動作時のログや、作成した地図がログフォルダ（例：2020_12_28_10_46_49_ee）として保存される
・doc------------------------TODOとか、その他
・include--------------------特になし
・io-------------------------ロボット動作中にロボットが表示するインターフェイス画像が入っている
・launch---------------------ROSローンチファイルが入っている。＜ファイル＞の.shファイルがそれらを参照してロボットを動かしている
・script---------------------ロボットプログラム（Python）が入っている
・src------------------------ロボットプログラム（C++）を入れるファイル

＜ファイル＞
・start_ee_robot.sh----------脳波実験（EEG experiment ; ee）用のロボットを動かすプログラム　インターフェイスが現れてロボットに搭載されたカメラ映像を見ることができる　ロボットをゲームパッドで操作してQテーブルを更新できる　使用方法は下記＜実行手順＞を参考のこと
・start_ee_sim.sh------------ee用の迷路シミュレータを動かすプログラム　使用方法は下記＜実行手順＞を参考のこと
・start_be_robot.sh----------行動実験（Behavioral experiment ; be）用のロボットを動かすプログラム　eeで作った地図およびQテーブルを参照して迷路探索する　eeのようなインターフェイスは出ない　ロボットは自律的に動く　Qテーブルの更新はされない　使用方法は下記＜実行手順＞を参考のこと
・start_be_sim.sh------------be用の迷路シミュレータを動かすプログラム　使用方法は下記＜実行手順＞を参考のこと
・CMakeLists.txt-------------おまじない　消すな
・package.xml----------------おまじない　消すな

＜実行手順＞
（地図作成及びQテーブル更新を行う場合）
１　PCにゲームパッドを接続する
２　bash start_ee_sim.sh {迷路の種類} {dataフォルダに保存したログフォルダの名前}（例：bash start_ee_sim.sh maze8 2020_12_28_10_46_49_ee）を端末で叩く
　　　{迷路の種類} = maze7 or maze8　←　~/ros_ws/res_ws/src/tb3_pkgs/turtlebot3_simulations/turtlebot3_gazebo/models/turtlebot3_mazeに迷路3Dモデルが入っている
３　GazeboとRvizの起動完了を待つ
４　新しい端末を開く
５　bash start_ee_robot.shを新しい端末で叩く
６　"Entar to START >>> "という文字が新しい端末に出てきたら、新しい端末でEnterを叩く
７　インターフェイスが出てくるのを待つ
８　ロボットが直進し始める
９　ロボットが交差点にたどりつたら、インターフェイスが進行方向を決定するよう指示してくるので、ゲームパッドに進行方向を入力する
１０　ロボットが交差点を移動する
１１　８〜１０をゴールにたどり着くまで繰り返し行う（終了）


（地図及びQテーブルを参照してロボットに自律移動させたい場合）
１　bash start_be_sim.sh {迷路の種類} {dataフォルダに保存したログフォルダの名前}（例：bash start_be_sim.sh maze8 exp）を端末で叩く
　　　{迷路の種類} = maze7 or maze8
　　　{dataフォルダに保存したログフォルダの名前} = ログフォルダの名前 or exp　←　expを指定すると、B4のeeで得られた地図とQテーブルを参照する
２　GazeboとRvizの起動完了を待つ
３　新しい端末を開く
４　bash start_be_robot.shを新しい端末で叩く
５　"Entar to START >>> "という文字が新しい端末に出てきたら、新しい端末でEnterを叩く
６　ロボットが動く（終了）
